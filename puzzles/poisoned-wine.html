Poisoned Wine Puzzle

A puzzle about using survival patterns to encode many possibilities at once.

Scenario

A king has 1,000 bottles of wine.

Exactly one bottle is poisoned.

The poison is colorless, odorless, and tasteless.

The king has 10 prisoners available to test the wine.

A prisoner who drinks poisoned wine will die exactly one day later. A prisoner who does not drink poisoned wine will survive.

Each prisoner may drink directly from any number of bottles, and each bottle may be sampled by any number of prisoners. Bottles may not be combined or altered before testing.

The king has time for only one round of testing before the wine must be served.

How can the king identify the poisoned bottle?
Suppose There Are Only 2 Bottles

There are only two bottles, one of which is poisoned.

You have one prisoner available for testing.

How can you determine which bottle is poisoned after one round of testing?
Return to the Original Problem

Now return to the original puzzle.

There are 1,000 bottles and 10 prisoners.

Each prisoner has only two possible outcomes after the test: they either survive or they do not.

If each additional prisoner also has only two possible outcomes, how much new information does each prisoner contribute?
Complete Explanation

With one prisoner and two bottles, the solution is straightforward: have the prisoner drink from one bottle.

  • If the prisoner survives, the other bottle must be poisoned.
  • If the prisoner dies, the bottle they drank from was poisoned.

One prisoner therefore provides two possible outcomes.

The key insight is that each additional prisoner doubles the number of different outcome patterns.

  • 1 prisoner → 2 outcomes
  • 2 prisoners → 4 outcomes
  • 3 prisoners → 8 outcomes
  • ...
  • 10 prisoners → 2¹⁰ = 1,024 outcomes

Since 1,024 is greater than 1,000, there are enough unique patterns to assign one to every bottle.

Giving Each Bottle a Unique Fingerprint

Instead of assigning one prisoner to each bottle, the king gives every bottle a unique identification code. That code determines which prisoners drink from that bottle.

Consider a smaller example with four bottles and two prisoners.

Bottle Code Prisoner 1 Drinks? Prisoner 2 Drinks?
Bottle 1 00 No No
Bottle 2 01 No Yes
Bottle 3 10 Yes No
Bottle 4 11 Yes Yes

Notice that the code is also a recipe for the tasting:

  • Prisoner 1 drinks whenever the first position contains a 1.
  • Prisoner 2 drinks whenever the second position contains a 1.

Every bottle therefore has its own unique fingerprint based on which prisoners drank from it.

Reading the Result

Suppose Bottle 3 is poisoned. Its code is 10.

Prisoner 1 drank from Bottle 3 and dies.

Prisoner 2 did not drink from Bottle 3 and survives.

Reading death as 1 and survival as 0, the resulting pattern is:

Prisoner 1: 1    Prisoner 2: 0
Resulting code: 10

That pattern matches the code assigned to Bottle 3, so the king can identify it as the poisoned bottle.

This identification system is called binary notation. The reader does not need to begin by understanding binary; the important idea is that every prisoner contributes one position to the bottle's fingerprint.

With ten prisoners, each bottle receives a unique ten-position code. A prisoner drinks from a bottle whenever their position in that bottle's code contains a 1.

After one day, the pattern of deaths and survivors reproduces the poisoned bottle's unique fingerprint. Since ten prisoners can create 1,024 different patterns, one poisoned bottle among 1,000 can be identified after only one round of testing.
